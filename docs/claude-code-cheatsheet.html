Install & Get Started

Pick your surface

Claude Code runs across terminal, VS Code, JetBrains, Desktop app, and the web. Every surface shares the same engine, so your CLAUDE.md, settings, and MCP servers travel with you.

Terminal — macOS / Linux / WSL

curl -fsSL https://claude.ai/install.sh | bash

Auto-updating native install. No Node.js required.

Terminal — Windows PowerShell

irm https://claude.ai/install.ps1 | iex

Requires Git for Windows. For CMD use curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd.

Homebrew

brew install --cask claude-code

claude-code tracks stable, claude-code@latest tracks latest. Homebrew does not auto-update — run brew upgrade.

WinGet

winget install Anthropic.ClaudeCode

Update with winget upgrade Anthropic.ClaudeCode.

VS Code / Cursor

Search "Claude Code" in Extensions (⌘⇧X / Ctrl+Shift+X), install, then ⌘⇧P → Claude Code: Open in New Tab. Inline diffs, @-mentions, plan review built in.

JetBrains IDEs

Install Claude Code from the JetBrains Marketplace. Works with IntelliJ, PyCharm, WebStorm, GoLand, RubyMine, and others.

Desktop App

macOS (Intel + Apple Silicon) and Windows (x64 + ARM64). Visual diff review, multiple parallel sessions, scheduled cloud tasks. Paid subscription required.

Web — claude.ai/code

Zero setup. Kick off long-running tasks, work on repos you don't have locally, pull sessions into your terminal later with claude --teleport.

First run cd your-project && claude — you'll be prompted to log in. For API-usage billing instead of a subscription, use claude auth login --console.

01.4CLAUDE.md — your project's persistent memory

CLAUDE.md sits in your project root and loads at the start of every session. Use it for coding standards, architecture decisions, preferred libraries, review checklists, and anything you'd otherwise have to repeat in every prompt.

Generate a starter:

Claude also builds auto memory as it works — saving build commands, debugging insights, and project quirks across sessions without you writing a thing. View or toggle with /memory.

Tip Nested CLAUDE.md files in subdirectories are loaded lazily when Claude accesses those folders. Use them to scope rules per subsystem — e.g. one for /backend, another for /frontend.

02.1Models & effort levels

Claude Code supports the full Claude 4 family. Effort levels trade latency for depth of reasoning.

ModelRole
claude-opus-4-7Flagship reasoning + agentic coding. 3× vision resolution. Default for complex work.
claude-opus-4-6Previous flagship. 1M context window GA at standard pricing.
claude-sonnet-4-6Balanced speed/intelligence. Default on Free + Pro. 1M context.
claude-haiku-4-5-20251001Fast + cheap. Great for subagents and high-volume pipelines.

Effort levels (Opus 4.7)

/effort high
claude --effort xhigh
  • low / medium / high — persist across sessions
  • xhigh — new in Opus 4.7, sits between high and max
  • max — current session only, Opus 4.6+ required
  • auto — reset to the model's default

Runtime shortcut: Option+P (macOS) or Alt+P (Win/Linux) to switch model without clearing your prompt.

03.1Subagents & agent teams

Subagents are isolated Claude instances with their own context, spawned by a lead agent. They're ideal for delegating bounded tasks — "find all API endpoints", "review this diff", "upgrade all deps".

Command / flagWhat it does
claude agentsList all configured subagents, grouped by source.
/agentsManage agent configurations interactively.
claude --agent my-custom-agentRun this session as a named agent.
claude --agents '{"reviewer": {...}}'Define dynamic subagents via JSON inline.
claude --teammate-mode in-processControl how agent-team teammates display: auto, in-process, tmux.

Agents live in .claude/agents/*.md with YAML frontmatter. They can define their own hooks, tools, and isolation: "worktree" for sandboxed work.

Example: an Explore-style custom agent

---
name: security-reviewer
description: Reviews diffs for security vulnerabilities
tools: Read, Grep, Glob, Bash
model: sonnet
isolation: worktree
hooks:
  PreToolUse:
    - matcher: "Bash"
      hooks:
        - type: command
          command: "./scripts/security-check.sh"
---

You are a security reviewer. Check for injection, auth
issues, secret exposure, and unsafe defaults.

03.2Hooks — the lifecycle API

Hooks are shell commands, HTTP endpoints, LLM prompts, or agents that fire at specific lifecycle events. The complete event list, grouped by cadence:

Per-session

  • SessionStart — new or resumed session. Load dev context, set env vars via CLAUDE_ENV_FILE.
  • SessionEnd — cleanup, telemetry, state save.
  • InstructionsLoaded — a CLAUDE.md or .claude/rules/*.md file entered context.

Per-turn

  • UserPromptSubmit — validate, inject context, block, or auto-title.
  • Stop — Claude finished responding. Block with decision: "block" to continue the turn.
  • StopFailure — turn ended from an API error (rate limit, auth, billing).
  • PreCompact / PostCompact — before/after context compaction.

Per-tool-call (inside the agentic loop)

  • PreToolUse — can block. Return permissionDecision: "allow" | "deny" | "ask" | "defer".
  • PermissionRequest — fires when a dialog is about to show. Allow/deny on the user's behalf.
  • PermissionDenied — auto-mode classifier denied. Return retry: true to let the model retry.
  • PostToolUse / PostToolUseFailure — after a tool succeeds / fails.

Agent team & teammate

  • SubagentStart / SubagentStop — when a subagent spawns / finishes.
  • TaskCreated / TaskCompleted — enforce task naming / completion gates.
  • TeammateIdle — quality gate before a teammate goes idle.

Environment & watchers

  • CwdChanged — react to cd. Write to CLAUDE_ENV_FILE to persist env.
  • FileChanged — watch literal filenames (e.g. .envrc|.env). Perfect for direnv-style flows.
  • ConfigChange — block/audit settings-file changes. Matcher filters source.
  • WorktreeCreate / WorktreeRemove — replace default git worktree behavior (SVN, Mercurial, Perforce).

MCP

  • Notification — permission_prompt, idle_prompt, auth_success, elicitation_dialog.
  • Elicitation / ElicitationResult — MCP server asks for user input mid-task. Intercept to respond programmatically.

Example — block destructive rm -rf

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "if": "Bash(rm *)",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/block-rm.sh"
          }
        ]
      }
    ]
  }
}

Example — async background hook

{
  "type": "command",
  "command": "./scripts/run-tests.sh",
  "async": true,
  "asyncRewake": true
}

asyncRewake: true runs the hook in the background and wakes Claude on exit code 2 — perfect for long test suites.

Prompt-based hooks (new)

{
  "type": "prompt",
  "prompt": "Given this tool call: $ARGUMENTS, does it leak PII? Reply yes/no JSON.",
  "model": "haiku"
}

03.3Skills — your own slash commands

A skill is a prompt handed to Claude. Drop a markdown file in .claude/skills/ (project) or ~/.claude/skills/ (personal), add frontmatter, and it shows up in the / menu. Claude can also invoke skills automatically when the description matches.

Bundled skills (built-in)

SkillWhat it does

/simplify [focus]Three parallel review agents aggregate findings, then fix code reuse/quality/efficiency issues.
/debug [description]Enable debug logging + troubleshoot from the session log.
/claude-apiLoad Claude API reference for your project's language. Auto-activates on anthropic import.
/loopRepeat a prompt while the session stays open. .claude/loop.md for the default.

Custom skill skeleton

---
name: review-pr
description: Review the current PR for my team's style rules
tools: Read, Grep, Bash
---

You are reviewing a pull request. Check for:
- TypeScript strict-null compliance
- Missing error handling on async functions
- Missing tests for public APIs

Output a markdown checklist of issues with file:line refs.

List available skills: /skills. Description listing cap is 1,536 chars (raised from 250) — you get space for real triggering nuance.

03.4Plugins & marketplaces

Plugins bundle skills, hooks, MCP servers, and agents into a single installable unit.

03.5MCP — connect external systems

The Model Context Protocol lets Claude read Google Drive, update Jira, query Slack, call internal APIs. Connect once, reuse everywhere.

claude mcp
/mcp

Load via config file

claude --mcp-config ./mcp.json
claude --strict-mcp-config --mcp-config ./mcp.json

MCP tools appear as mcp__<server>__<tool>. Match every tool from one server in hooks with mcp__memory__.*. MCP servers can also expose prompts as /mcp__<server>__<prompt>.

New: MCP OAuth 9728 support, 500K result persistence, conditional filtering for hooks.

03.7Headless mode & the Agent SDK

Print mode (-p) is the foundation of the Agent SDK. Combine it with structured outputs and stream-json to build production agents.

claude -p "list all TODOs as JSON" --output-format json
claude -p --json-schema '{"type":"object","properties":{"issues":{"type":"array"}}}' "find bugs"
claude -p --input-format stream-json --output-format stream-json --include-hook-events "query"

Flags that matter for scripting

  • --bare — skip auto-discovery for fast scripted calls.
  • --no-session-persistence — don't write sessions to disk (print mode only).
  • --exclude-dynamic-system-prompt-sections — move per-machine sections out of the system prompt so prompt-cache reuses across users/machines.
  • --permission-prompt-tool mcp_auth_tool — route permission prompts through an MCP tool in non-interactive mode.
  • --session-id <uuid> — pin a specific session ID.
  • --replay-user-messages — re-emit user messages back on stdout for ack (stream-json in + out).
New in 2.1.89+ "defer" as a PreToolUse permission decision lets SDK apps pause at a tool call, collect input via their own UI, and resume with claude -p --resume <id>. The pending call is preserved in the transcript with stop_reason: "tool_deferred".
